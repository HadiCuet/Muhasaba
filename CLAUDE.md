## Commands

- Install/refresh dependencies: `flutter pub get`
- Run the app (choose a device with `-d`): `flutter run`
- Static analysis / lint: `flutter analyze`
- Format: `dart format .`
- Run all tests: `flutter test`
- Run a single test file: `flutter test test/widget_test.dart`
- Run a single test by name: `flutter test --plain-name "Counter increments smoke test"`
- Build release artifacts: `flutter build apk` / `flutter build ios` / `flutter build web` / `flutter build macos`

Dart SDK constraint: `^3.11.4` (see `pubspec.yaml`).

## Architecture

This repo is currently the **default Flutter `flutter create` starter**, effectively untouched:

- Entry point: `lib/main.dart` — `main()` → `MyApp` (`MaterialApp`) → `MyHomePage` (stateful counter demo).
- No state-management library, router, DI, networking layer, or feature modules have been introduced yet. `lib/` contains only `main.dart`.
- Tests live in `test/` and use `flutter_test`; `test/widget_test.dart` is the scaffolded counter smoke test.
- Lints come from `package:flutter_lints/flutter.yaml` via `analysis_options.yaml`; no custom rules are enabled.
- Platform shells under `android/`, `ios/`, `macos/`, `linux/`, `windows/`, `web/` are the defaults generated by `flutter create`.

When adding the first real feature, establish the architectural choices (state management, routing, folder layout) deliberately — there is no existing pattern in `lib/` to follow.
